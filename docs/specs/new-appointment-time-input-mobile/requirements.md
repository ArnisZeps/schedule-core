# Requirements: New Appointment Time Input Mobile

## User stories

- As a staff member on an iPhone, I want to enter a custom start time in the new appointment panel without the input overflowing its container, so that the form is usable on mobile.

## Acceptance criteria

- [ ] On Safari iOS, the custom time input does not visually overflow the "Start time" border container.
- [ ] On Chromium (desktop and device-emulation), the custom time input renders identically to before.
- [ ] The "Ends at HH:MM" computed end time continues to display correctly after time entry.
- [ ] Form submission with a custom time continues to work as before.
