# Requirements: calendar-local-nav

## User stories

- As a business owner, I want clicking prev/next week and switching views to feel instant, so that browsing the schedule is not frustrating.

## Acceptance criteria

- [ ] Clicking prev/next/today updates the calendar immediately with no perceptible delay
- [ ] Switching week/day/list view updates immediately
- [ ] Changing the service or staff filter updates immediately
- [ ] First load from a direct URL (e.g. `/appointments?date=2026-05-26`) initialises to that date
- [ ] Browser back/forward button navigates between previously viewed dates
- [ ] Existing tests remain green
