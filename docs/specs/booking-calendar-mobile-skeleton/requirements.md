# Requirements: Booking Calendar Mobile Skeleton

## User stories

- As a mobile client, I want the loading skeleton in the booking calendar to look like the actual calendar, so that I get an accurate preview of the content about to appear.

## Acceptance criteria

- [ ] On mobile viewports, the skeleton calendar matches the visual footprint of the real calendar (same number of rows, same cell proportions).
- [ ] On desktop viewports, the skeleton is unchanged in appearance.
- [ ] No skeleton cells protrude outside the calendar's visual boundary during loading.
- [ ] Skeleton cells transition to real calendar cells without any layout shift.
